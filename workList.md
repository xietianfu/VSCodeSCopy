# VS Code Easy Copy 插件 - 最新开发任务拆解

基于架构师与产品经理的讨论共识，整合后的方案如下：

***

## 一、MVP 开发计划（49h，约 6 个工作日）

### 阶段一：项目初始化 + 基础架构（10h）

#### Task 1.1: 项目脚手架搭建

- 使用 `yo code` 生成 VS Code 扩展项目
- 配置 TypeScript + ESLint + Prettier
- 设置开发环境（编译、调试配置）
- **工时：4h**

#### Task 1.2: 核心数据结构定义

- 创建 `types.ts` 定义 `CodeBlock`、`Collection` 接口
- 创建 `ExtensionState` 管理全局状态
- 创建 `StorageService` 封装 globalState 操作
- **工时：3h**

#### Task 1.3: 项目识别服务 + 存储验证

- 创建 `ProjectService` 获取当前 workspace folder
- 处理未保存文件的命名（Untitled-{n}）
- **新增：验证 globalState 存储限制（目标 2MB+）**
- **工时：3h**

***

### 阶段二：核心收集功能（12h）

#### Task 2.1: 代码块收集命令

- 实现 `easy-copy.collect` 命令
- 获取当前选中内容的位置（文件路径、行号范围）
- 处理未选中代码的情况
- **工时：4h**

#### Task 2.2: 右键菜单 + 快捷键集成

- 在 `package.json` 配置 `menus` 和 `keybindings`
- 添加快捷键 `Cmd+;` 绑定
- **新增：验证不同键盘布局的兼容性（US/德语等）**
- **工时：3h**

#### Task 2.3: 去重逻辑（基础版）

- 完全重复检测：同文件同行号 → 跳过
- 部分重叠合并：扩展行号范围
- Toast 反馈提示
- **工时：5h**

***

### 阶段三：浮动面板 Webview（12h）

#### Task 3.1: Webview 基础架构

- 创建 `CollectionPanel` 类管理 Webview
- 实现消息通信协议（collect/delete/update）
- **工时：3h**

#### Task 3.2: 面板 UI 实现（低保真）

- 使用原生 HTML/CSS/JS 构建
- 代码块列表展示（色块 + 文件名 + 行号 + 删除按钮）
- AI 提示输入框
- 底部操作按钮（完成并复制、暂存）
- **工时：6h**

#### Task 3.3: 快捷键绑定

- `Cmd+Enter` → 完成并复制
- `Esc` → 暂存
- `Tab` → 聚焦输入框
- **工时：2h**

#### Task 3.4: 基础样式适配

- VS Code 原生主题适配
- 基础动画效果
- **工时：1h**

***

### 阶段四：复制与暂存功能（5h）

#### Task 4.1: 复制功能实现

- 格式化输出：AI 提示 + 代码位置列表
- VS Code Clipboard API 写入
- 成功反馈：「✓ 已复制」→ 3s 倒计时 → 关闭
- **工时：3h**

#### Task 4.2: 暂存功能

- 保存到 globalState
- 下次打开恢复内容
- **工时：2h**

***

### 阶段五：状态栏徽章（2h）

#### Task 5.1: 状态栏集成

- 创建 `StatusBarItem` 显示暂存数量
- 点击打开浮动面板
- **工时：2h**

***

### 阶段六：核心功能测试（8h）

#### Task 6.1: 单元测试

- 核心逻辑测试（去重、合并、项目识别）
- 覆盖率目标：60%+
- **工时：4h**

#### Task 6.2: 手动测试

- 所有边界情况验证
- 不同操作系统测试（macOS/Windows/Linux）
- **工时：4h**

***

## 三、技术栈确认

- **语言**: TypeScript
- **框架**: VS Code Extension API
- **UI**: 原生 HTML/CSS（避免框架依赖）
- **测试**: Jest/Mocha
- **构建**: webpack/vscode-loader

***

## 四、依赖关系

```
阶段一 → 阶段二 → 阶段三 → 阶段四 → 阶段六
                                ↓
                            阶段五
```

阶段五（状态栏）可与阶段四并行开发。

***

## 五、风险点与应对

| 风险         | 应对措施                 |
| :--------- | :------------------- |
| 快捷键冲突      | 阶段二验证多键盘布局，预留可配置方案   |
| 存储限制       | 阶段一验证 globalState 上限 |
| Webview 性能 | 暂不需要虚拟滚动，等用户反馈       |

***

## 六、验收标准（MVP）

- 快捷键 `Cmd+;` 可触发收集
- 右键菜单显示「+ 收集」选项
- 浮动面板正确显示已收集的代码块
- 同文件多段用相同色块，不同文件用不同色块
- 输入 AI 提示后，复制内容包含提示
- 点击「完成并复制」后，剪贴板内容格式正确
- 点击「暂存」后，内容保留，下次添加恢复
- 状态栏显示暂存数量
- 去重逻辑正确处理重复和重叠
- Toast 反馈及时清晰
