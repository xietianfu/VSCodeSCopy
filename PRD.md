# Easy Copy 插件 PRD v1.0

## 一、产品概述

### 1.1 产品定位
VS Code 插件，解决 AI CLI 编程场景下的代码审查反馈效率问题。开发者可在 VS Code 中快速收集多处代码位置（文件路径+行号范围），批量反馈给 AI CLI 进行精准修复。

### 1.2 目标用户
- 使用 AI CLI（如 Aider、Cursor Continue、自研 CLI）进行开发的开发者
- 每天需要进行多次代码审查，并将问题反馈给 AI 的场景

### 1.3 核心价值
| 现状 | 使用后 |
|------|--------|
| 手动复制路径+行号，每次 30 秒 | 快捷键一键收集，每次 2 秒 |
| 分散反馈，容易遗漏 | 批量收集，完整反馈 |
| 心流频繁打断 | 专注审查，最后统一反馈 |

## 二、功能规格

### 2.1 核心功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 代码块收集 | P0 | 选中代码后添加到浮动面板 |
| 浮动面板 | P0 | 实时显示已收集的代码块 |
| AI 提示输入 | P0 | 可选输入问题描述 |
| 完成并复制 | P0 | 一键复制所有内容到剪贴板 |
| 暂存 | P0 | 关闭面板，保留内容，下次继续 |
| 历史记录 | P0 | 侧边栏查看历史记录，支持再次复制 |
| 去重合并 | P0 | 自动处理重复和重叠的行号范围 |
| 状态栏徽章 | P1 | 显示暂存数量 |
| 配置项 | P1 | 自定义快捷键、色块、历史上限 |

### 2.2 输出格式

```
{AI 提示内容（可选）}

{文件名}:{行号范围}
{文件名}:{行号范围}
...
```

**示例：**
```
请修复以上代码的类型错误

Header.tsx:20-30
Header.tsx:50-80
api.ts:15-22
```

## 三、交互设计

### 3.1 触发方式

| 方式 | 操作 |
|------|------|
| 快捷键 | `Cmd+;`（macOS）/ `Ctrl+;`（Windows/Linux） |
| 右键菜单 | 编辑器右键 → 「+ 收集」 |

### 3.2 浮动面板交互

```
┌─────────────────────────────────────────────────────┐
│ 📋 已收集 2 个代码块                           [×]    │
├─────────────────────────────────────────────────────┤
│ 🟦 Header.tsx                             [×]        │
│    20-30                                                │
│    50-80                                                │
├─────────────────────────────────────────────────────┤
│ 🟧 api.ts                                 [×]        │
│    15-22                                                │
├─────────────────────────────────────────────────────┤
│ 💬 给 AI 的提示（可选）                               │
│ ┌─────────────────────────────────────────────────┐ │
│ │ 例如：修复类型错误，检查边界情况...                 │ │
│ └─────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────┤
│          [暂存]              [✓ 完成并复制]            │
└─────────────────────────────────────────────────────┘
```

**交互逻辑：**
- 首次添加 → 浮动面板弹出（聚焦输入框）
- 继续添加 → 面板保持打开，实时更新
- 点击「暂存」→ 立即关闭，保留内容
- 点击「完成并复制」→ 复制 + 显示「✓ 已复制」→ 3 秒倒计时 → 关闭

**快捷键（面板内）：**
| 按键 | 动作 |
|------|------|
| `Cmd+Enter` | 完成并复制 |
| `Esc` | 暂存（关闭，保留） |
| `Tab` | 聚焦输入框 |

### 3.3 状态栏交互

```
状态栏右侧：📋 3
```

- 数字表示暂存的代码块数量
- 点击徽章 → 快速打开暂存的浮窗

### 3.4 侧边栏历史记录

```
┌─────────────────────────────────────────────────────┐
│ 📋 代码审查历史                        [🔍 搜索]    │
├─────────────────────────────────────────────────────┤
│ ▶ 项目 A (my-app)                                   │
│   今天 16:05                                         │
│   ┌─────────────────────────────────────────────┐   │
│   │ 💬 请修复以上代码的类型错误...                │   │
│   │ Header.tsx:20-30, 50-80                       │   │
│   │ api.ts:15-22                                  │   │
│   │                          [再次复制] [删除]    │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
│   今天 14:30                                         │
│   ┌─────────────────────────────────────────────┐   │
│   │ 💬 检查这里的逻辑问题                         │   │
│   │ utils.ts:100-120                             │   │
│   │                          [再次复制] [删除]    │   │
│   └─────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│ ▶ 项目 B (another-project)                          │
│   昨天 09:15                                         │
│   ┌─────────────────────────────────────────────┐   │
│   │ Button.tsx:45-52                             │   │
```

**历史记录规则：**
- 按项目分组（可折叠）
- 同项目内按时间倒序（最新在上）
- 每个项目显示最近记录，点击展开查看更多
- 全局上限 50 条，超出自动删除最早

## 四、视觉设计

### 4.1 色块方案

同一文件的多段代码用相同色块标识，不同文件用不同颜色：

| 文件序号 | 色块 | HEX | 辅助标识 |
|---------|------|-----|----------|
| 1 | 🟦 | `#4A90E2` | 数字 1 |
| 2 | 🟧 | `#F5A623` | 数字 2 |
| 3 | 🟩 | `#7ED321` | 数字 3 |
| 4 | 🟪 | `#BD10E0` | 数字 4 |
| 5+ | 🟫 | `#8B572A` | 数字序号 |

### 4.2 反馈动画

| 场景 | 反馈 |
|------|------|
| 添加成功 | Toast：「已收集 Header.tsx:20-30」 |
| 重复添加 | Toast：「已存在，已跳过」 + 已有项闪烁 |
| 复制成功 | 面板底部：「✓ 已复制」→ 3 秒倒计时动画 |
| 合并行号 | Toast：「已合并为 20-35」 |

## 五、技术实现要点

### 5.1 数据结构

```typescript
interface CodeBlock {
  filePath: string;        // 相对路径
  fileName: string;        // 文件名
  startLine: number;       // 起始行
  endLine: number;         // 结束行
  colorIndex: number;      // 色块索引
  projectId: string;       // 项目标识
}

interface Collection {
  id: string;
  blocks: CodeBlock[];
  prompt?: string;         // 用户输入的问题
  createdAt: Date;
  projectId: string;
  projectName: string;
}

interface HistoryRecord extends Collection {
  copied: boolean;
}
```

### 5.2 去重逻辑

| 场景 | 处理 |
|------|------|
| 完全重复（同文件同行号） | 跳过，Toast 提示 |
| 包含重复（新选段在已有段内） | 跳过，提示 |
| 部分重叠 | 合并为更大的范围 |

### 5.3 持久化策略

| 数据 | 存储位置 | 生命周期 |
|------|----------|----------|
| 当前暂存 | globalState | 跨会话保留 |
| 历史记录 | globalState | 跨会话保留，上限 50 条 |
| 用户配置 | workspaceConfiguration | 工作区级别 |

### 5.4 项目识别

- VS Code workspace folder URI 作为项目 ID
- 文件路径相对于 workspace folder
- 未保存文件使用 `Untitled-{n}`

## 六、边界情况处理

| 场景 | 处理 |
|------|------|
| 未选中代码 | Toast：「请先选中代码」 |
| 未保存的新文件 | 使用 `Untitled-1:20-30` + 警告图标 |
| 只读文件 | 正常处理（路径有效） |
| 选中单行 | 输出 `src/file.ts:45`（单行） |
| 多段非连续选中 | 分别添加到清单 |
| 历史记录已满 | 自动删除最早的记录 |
| 跨项目使用 | 按项目分组显示 |

## 七、配置项

在 VS Code 设置中新增 `easy-copy.*` 配置项：

```json
{
  "easy-copy.shortcut": "Cmd+;",
  "easy-copy.historyLimit": 50,
  "easy-copy.colors": ["#4A90E2", "#F5A623", "#7ED321", "#BD10E0", "#8B572A"],
  "easy-copy.promptPlaceholder": "例如：修复类型错误，检查边界情况...",
  "easy-copy.showWelcome": true
}
```
